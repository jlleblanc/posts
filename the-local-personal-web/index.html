The 2010s did not merely introduce shared cloud platforms. It made them an everyday part of the way people shop, work, travel, and socialize. Cloud platforms in turn suburbanized the web, separating each of these activities into single-use properties shared among millions of strangers.

People are beginning to speak up about the abuses of Big Tech cloud platforms. Governments are passing legislation in an attempt to regulate their activity. The Internet itself is getting slower with so many people working and learning remotely.

After a decade of cloud platform dominance, we are ready for something new. Here at the beginning of the 2020s, we have an opportunity to swing back to personal computing power.

Local

Think of a city on a different continent from the one where you're reading this. Now think of the nearest family-run restaurant to you. Does that restaurant have a website? And if it does, does this website actually need to be available in that city on a different continent?

Chances are, if the website were only available within a 30 mile radius of the restaurant, it would suffer little to no loss of business. It may be interesting on some level to be able to load up a website for a restaurant a continent away, but the economic benefit is close to nil.

And yet the platforms we use for publishing to the World Wide Web are designed so our sites load quickly regardless of where someone is in the world. We use content delivery networks, cloud hosting, and a domain name that hasn't already been taken. Even if the majority of our audience lives nearby, our website could be hosted on a server several timezones away.

With the Local Personal Web, instead of paying to host on servers nowhere nearby, local businesses will host websites on site using hardware they already own. The server could be hosted on a desktop computer, a mobile device, or even the router.

Personal

If you're not a web developer or software engineer, you probably don't have a personal website. With the social media networks at your disposal, why would you?

Web

Perhaps surprisingly, the Web is designed for the long haul. It certainly hasn't always been implemented that way. However, the standards-stewarding organizations responsible for the Web have insisted on backwards compatibility and progressive enhancement. These principles have made it possible to adapt the Web to new uses without breaking existing sites.

We can reuse Web technology in a new context by merging it with the phone system.
